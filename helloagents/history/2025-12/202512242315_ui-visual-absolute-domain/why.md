# 变更提案: UI 视觉绝对领域迭代与全量细节审判

## 需求背景

项目当前已经具备较强的架构自洽与基础视觉统一，但仍存在“可感知的细节差异”，主要集中在：

1. **排版与对齐一致性**：不同组件间的间距、字号、行高与对齐规则存在差异，影响“像素级”观感。
2. **交互触感**：按钮、弹窗、列表进入动画等仍使用较通用的 `ease` 过渡曲线，缺少高级触感与统一的动效语言。
3. **暗黑模式细节**：虽然已有 light/dark 主题，但对比度层级、渐变/噪点强度、滚动/焦点态等细节仍可进一步精炼。
4. **低级错误与可访问性**：潜在的 console warning、ARIA 细节、失效 label、非标准嵌套与拼写瑕疵需要地毯式清理。

本次迭代目标是把 UI 从“好看”提升到“商业软件级细节密度”，并建立可持续迭代的视觉与交互基线。

## 变更内容

1. **排版与间距系统统一**
   - 全局引入可复用的 spacing/typography/easing token（CSS 变量）
   - 修正各页面标题、卡片、表格、表单的行高与对齐规则
2. **动效与触感升级**
   - 引入自定义 cubic-bezier 曲线（更接近 spring 的感知）
   - 统一按钮/弹窗/列表的进入与点击反馈，并支持 `prefers-reduced-motion`
3. **暗黑模式细节打磨**
   - 优化对比度与背景噪点强度、统一 focus ring 与 hover 层级
   - 长文本截断与渐变遮罩（避免硬切）
4. **全量低级错误扫描**
   - ARIA/语义标签修复、失效属性清理、潜在警告消除

## 影响范围

- 全局样式：`src/style.css`
- 基础组件：`src/components/base/*`
- 通用组件：Header/Toast/Modal 等
- 页面与业务组件：视具体问题点逐步修正

## 风险评估

- **风险:** 过度“视觉统一”可能改变现有页面的局部布局  
  **缓解:** 优先以 token + 小范围修正方式推进，不做破坏性重排。
- **风险:** 动效过强影响可访问性/性能  
  **缓解:** 支持 `prefers-reduced-motion`，并避免持续性高开销动画。
